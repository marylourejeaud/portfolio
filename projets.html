Simulation ETL "Dilemme du Prisonnier"

Conception d'un pipeline ETL complet autour d'une simulation multi-stratégies.

+
Python Parquet Streamlit IA Locale
  • Génération des données : création d'un moteur de simulation impliquant l'IA ; enregistrement des choix, scores cumulés et justifications textuelles.
  • Transformation : nettoyage, structuration et enrichissement du dataset (indicateurs de coopération, fréquence de trahison, évolution des scores, dérivations par stratégie).
  • Analyse avancée : regroupement des justifications par stratégie et analyse comportementale automatisée via IA locale ; export en CSV.
  • Visualisation : développement d'une application Streamlit présentant courbes temporelles, heatmaps, comparaisons de stratégies et explorateur de justifications IA.
  • Stockage optimisé : création d'un dataset Parquet propre et analysable pour l'ensemble des simulations.

Pipeline ETL Spotify Million Playlist

Conception d'un pipeline complet pour traiter un volume massif de données JSON (1M+ playlists).

+
Python (Multiprocessing) PostgreSQL Polars
  • Ingestion des fichiers : utilisation de Python et multiprocessing pour le parsing structuré et l'extraction des entités (playlists, tracks, artists, albums).
  • Modélisation relationnelle : création d'un schéma PostgreSQL optimisé.
  • Chargement rapide : utilisation de buffers en mémoire et d'une insertion en bloc, avec gestion automatique des doublons.
  • Transformation : traitement final et export en fichiers Parquet via Polars.

Benchmark ETL & Modèles IA

Conception d'un pipeline de data engineering et data visualisation pour comparer la fiabilité des modèles d'IA.

+
Pandas Plotly Express Ollama API
  • Collecte automatisée : requêtes via API Open Trivia Database et intégration des réponses de modèles d'IA (LLMS - Ollama).
  • Enrichissement du dataset : analyse de performance des modèles (taux de bonnes réponses, rapidité, précision par catégorie/difficulté).
  • Reporting : génération d'un rapport visuel interactif (histogrammes, heatmaps, courbes de temps de réponse).
